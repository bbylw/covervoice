# 质量检查清单（Checklist）

这个清单来自真实短视频封面迭代过程。每一条都是踩过坑之后总结的，按重要性排序。

生成封面后，逐项对照。

---

## 🔴 P0 · 一定不能犯的错

### 0. 生成前必须通过的类名校验（最重要）

**现象**：直接把 layouts.md 的骨架粘到 HTML，结果样式全部丢失——大标题变成非衬线、标签挤成一团、图片不显示。

**根因**：如果 `template.html` 的 `<style>` 里没有这些类的定义，浏览器就 fallback 到默认样式。

**做法**：
- **生成前，必须先 `Read` `assets/template.html`**，确认 layouts.md 里用到的类都已定义
- 最常见遗漏的类：`h-cover` / `h-cover-en` / `h-sub` / `lead` / `body-cover` / `meta` / `kicker` / `tag` / `tag-fill` / `tag-row` / `accent-line` / `callout` / `author-row` / `meta-row` / `big-num` / `mid-num` / `frame-img` / `visual-area`
- 如果某个类确实缺了，**在 template.html 的 `<style>` 里补上**，不要 inline 重写
- 生成后打开浏览器，如果看到"大标题是非衬线"，几乎 100% 是这个问题

### 1. 主标题字数超限

**现象**：中文主标题写了 12 个字，竖版下挤成 3-4 行，每行 2-3 个字，极其难看。

**做法**：
- **中文主标题 ≤ 8 字，最佳 3-6 字**
- 超过 8 字必须精简或换布局
- 两行标题时，每行 ≤ 5 字，用 `<br>` 手工断行
- 绝对禁止出现"一行只有 1 个字"的情况

**自检**：在浏览器中打开，目测标题行数和每行字数。

### 2. 不要用 emoji 作图标

**现象**：在杂志风格封面里用 emoji（🎯 💡 ✅）会立刻破坏格调。

**做法**：用 Lucide 图标库：

```html
<i data-lucide="target" class="ico-md"></i>
```

常用图标名：`target` / `palette` / `compass` / `share-2` / `play` / `mic` / `book-open` / `trending-up`

### 3. 背景与文字对比度不足

**现象**：暗色背景图 + 白色文字，但背景图某区域很亮，导致白色文字看不清；或浅色背景 + 黑色文字，但背景图有深色区域。

**做法**：
- **暗色主题**：背景图必须是整体偏暗的，或 `#cover::before` 遮罩足够厚（默认 60%）
- **浅色主题**：背景图必须是整体偏亮的，或遮罩足够厚（默认 70%）
- 如果背景图明暗不均，**宁可加厚遮罩牺牲背景细节，也要保证文字 100% 可读**
- 图文叠加布局（Layout F）尤其要注意这一点

**自检**：把浏览器缩放到很小（模拟手机缩略图），检查文字是否仍然清晰可读。

### 4. 9:16 比例被篡改

**现象**：改了 `#cover` 的 width/height，或加了 padding/margin，导致比例不是 9:16。

**做法**：
- `#cover` 必须是 `width:1080px; height:1920px`
- 不要改这两个数字
- 不要给 `#cover` 加 border 或 padding（会破坏截图尺寸）
- 预览缩放靠外层 `#preview-wrap` 的 `transform:scale()`，不要动 `#cover`

### 5. 字体分工错乱

**做法**：
- 主标题 → **衬线**（Noto Serif SC / Playfair Display）
- 副标题/正文 → **衬线或非衬线**（根据层级）
- 标签/kicker/作者 → **等宽**（IBM Plex Mono）
- 数字 → **衬线英文**（Playfair Display）

严禁主标题用非衬线、严禁标签用非等宽。

## 🟡 P1 · 排版节奏

### 6. 信息层级清晰

封面只有一张，信息层级必须一眼可辨：

```
Layer 1（最大）: 主标题 .h-cover
Layer 2（中）  : 副标题 .h-sub 或 .lead
Layer 3（小）  : kicker / tag / 标签
Layer 4（最小）: 作者 / 日期 / 平台
```

**不要出现两层字号接近的情况**——比如主标题 112px，副标题 100px，读者无法判断哪个更重要。

### 7. 留白充足

**现象**：内容从顶部塞到底部，没有呼吸空间，像 Word 文档。

**做法**：
- `.content` 默认 `padding:90px 78px 110px`，上下左右都有安全边距
- mid-zone 的 `gap` 默认 36px，不要太密
- 标题和副标题之间至少留一个 `accent-line` 或 24px 间距
- 底部 `.bottom-zone` 不要太贴近边缘（已预留 110px）

### 8. kicker 和标题不要重复

**现象**：kicker 写"AI 工作流教程"，标题又写"三步搭建 AI 工作流"——信息重复。

**做法**：
- **kicker = 系列/栏目标签**（跨封面可复用）："Vol.01" / "EP.12" / "TECH TALK"
- **标题 = 封面独有的核心信息**："三步搭建 AI 工作流"
- 两者互为补充，不互相翻译

### 9. 标签数量控制

**现象**：顶部放了 5 个标签，换行两行，拥挤不堪。

**做法**：
- **标签 ≤ 3 个**
- 每个标签字数 ≤ 12 字符
- 如果标签多，改用 `tag-fill`（实心胶囊）交替使用，制造节奏

## 🟢 P2 · 视觉打磨

### 10. WebGL 背景在暗色主题下可见

暗色主题 + `hero` 类时，WebGL 背景应该明显透出，形成氛围。如果看起来只是一片黑：
- 检查 canvas 是否正确渲染（浏览器控制台看有无 WebGL 错误）
- 检查 `#cover` 是否有 `dark` 类
- 检查 shader 是否加载成功

### 11. 浅色主题下 WebGL 不过强

浅色主题下，WebGL 通过 CSS 自动降到 35% opacity。如果仍然太抢眼：
- 给 `#cover` 加 inline style 进一步降低：`style="--bg-opacity:0.2"`
- 或在 template.html 的 CSS 里调 `#cover.light canvas.bg` 的 opacity

### 12. 图片不要带文字

背景图或视觉图如果自带文字/水印/Logo，会和封面标题冲突，造成视觉混乱。

**做法**：
- 背景图必须是"纯视觉"——纹理、渐变、风景、人物、抽象图形
- 如果原图有文字，用无文字版本，或换图
- 产品截图做背景时，必须用高斯模糊或深色 overlay 压暗

### 13. 入场动画 stagger 合理

**现象**：所有元素同时出现，或延迟太久导致用户以为页面卡了。

**做法**：
- 按信息重要程度分配 `anim-dN`：
  - kicker / tag → `anim-d1`（最先出现）
  - 主标题 → `anim-d2`
  - 副标题 / accent-line → `anim-d3` ~ `anim-d4`
  - 底部信息 → `anim-d5` ~ `anim-d6`（最后出现）
- 不要给所有元素加同样的 delay

### 14. 圆角和边框克制

- 图片圆角固定 6px，**不要超过 8px**
- 不要加 `box-shadow`
- tag 的边框 1.5px，**不要加粗**
- callout 左边框 4px，**不要加粗**

## 🔵 P3 · 操作细节

### 15. 图片路径用相对路径

图片放在 `images/` 文件夹下，HTML 里用 `images/xxx.png`，不要用绝对路径。

### 16. 导出时隐藏提示条

`#export-hint` 是预览用的提示条，正式截图前可以临时隐藏：

```css
#export-hint{display:none}
```

或在 DevTools 里删掉这个元素再截图。

### 17. 截图分辨率

封面画布固定 1080 × 1920px，截图时必须保持这个分辨率：
- 浏览器截图插件：选择"截取完整尺寸"
- 系统截图：确保截取区域精确匹配封面边缘
- 不要截到浏览器边框、滚动条、或外部 body 的黑色背景

## 🎙 P0-Audio · TTS 配音检查（Step 7 必查）

### A1. 朗读文本与封面文案对齐

**做法**：
- 朗读文本来自封面 `lead` / `callout` / 主标题+副标题拼接，**不要凭空再写一段**
- 朗读时长建议 8-40 秒（30-120 字）
- 不能把英文单词夹在中文里读半英半中——TTS 引擎容易出错；要么全中文，要么明显的英文术语单独切分

### A2. 语音匹配内容调性

| 内容类型 | 推荐 voice | 理由 |
|---|---|---|
| 新闻 / 资讯 / 财经 | `zh-CN-YunjianNeural` | 男声·清晰稳重 |
| 知识 / 干货 / 教程 | `zh-CN-YunxiNeural` | 男声·沉稳可信 |
| 生活 / 种草 / 轻量 | `zh-CN-XiaoyiNeural` | 女声·活泼亲切 |
| 温柔 / 文学 / 情感 | `zh-CN-XiaoxiaoNeural` | 女声·温柔清晰 |
| 英文 / 国际化 | `en-US-SteffanNeural` | 美音男声·默认 |

### A3. 语速与音量

- 默认 `--rate "+0%"` `--volume "+0%"`,不要无缘无故改
- 新闻/快讯可 `+10%`,沉稳长文可 `-10%`
- 范围必须在 `±50%` 内,超出 edge-tts 会拒绝

### A4. 输出验证

```
□ voice.mp3 文件存在,大小 > 0
□ 用播放器试听一遍,无吞字 / 无杂音 / 标点停顿合理
□ 中文里的数字、英文缩写按预期发音(如 "AI" 读 "A I" 而非 "啊一")
```

如失败:重新跑一次(通常是网络问题),仍失败检查 voice 名称是否拼对。

## 🎬 P0-Video · 视频合成检查(Step 8 必查)

### V1. 输入文件

- `--image`:1080 × 1920 PNG/JPG,**不能是含浏览器边框的截图**
- `--audio`:Step 7 生成的 MP3,确保和封面文案一致

### V2. FFmpeg 依赖

```
□ ffmpeg --version 可执行
□ ffprobe --version 可执行(脚本会同时检查这两个)
```

### V3. 淡入淡出

- `--fade-in`:推荐 0.8 ~ 1.5 秒
- `--fade-out`:推荐 1.5 ~ 2.5 秒(短视频结尾留出"喘息感")
- 总淡出 + 淡入 不要超过音频时长的一半,脚本会自动夹取

### V4. 输出验证

```
□ video.mp4 文件存在,文件大小通常 0.3-3 MB(取决于音频长度)
□ 用播放器打开,分辨率精确 1080 x 1920
□ 时长 = 音频时长(由 -shortest 保证)
□ 首帧、末帧的封面无裁切、无黑边
□ 音画无错位、无延迟、无丢帧
```

### V5. 平台规格(可选)

| 平台 | 时长上限 | 备注 |
|---|---|---|
| 抖音 / TikTok | 60 秒(常规)/ 3 分钟(长) | 文件 < 100 MB |
| 小红书 | 15 分钟 | 9:16 优先 |
| B 站竖屏 | 不限 | 首屏 3 秒决定完播率 |
| 视频号 | 60 分钟 | 推荐 < 60 秒 |

## 🧪 最终自检清单

生成完封面 + 配音 + 视频后,逐项对照:

```
预检(生成前)
  □ 已读过 template.html 的 <style>,确认所需类都存在
  □ 已决定用哪个 Layout(A-F)
  □ 已选定主题色(5 套之一)
  □ <title> 已改为实际封面标题(grep "[必填]" 应无结果)

封面 · 内容
  □ 主标题 ≤ 8 字,没有一行只留 1 个字
  □ kicker 和标题不重复
  □ 副标题制造好奇缺口,不剧透
  □ 标签 ≤ 3 个
  □ 术语用法统一

封面 · 排版
  □ 信息层级清晰(标题 > 副标题 > 标签 > 作者)
  □ 留白充足,不拥挤
  □ 衬线 / 非衬线 / 等宽字体分工正确
  □ 有入场动画 stagger(不是同时出现)

封面 · 视觉
  □ 背景与文字对比度足够(手机缩略图也能看清)
  □ 背景图不带文字 / 水印 / Logo
  □ 圆角和边框克制
  □ 9:16 比例正确(1080 x 1920)

封面 · 导出
  □ 浏览器中打开正常
  □ WebGL 背景渲染正常(暗色主题下可见流体)
  □ 截图分辨率精确为 1080 x 1920
  □ 没有截到浏览器边框或 #export-hint

配音(Step 7)
  □ 朗读文本与封面文案语义一致
  □ voice 选型匹配内容调性
  □ 语速 / 音量在 ±50% 内
  □ voice.mp3 试听无吞字、无杂音
  □ 数字 / 英文缩写按预期发音

视频(Step 8)
  □ 输入图为干净的 1080 x 1920 PNG/JPG
  □ ffmpeg 与 ffprobe 都在 PATH
  □ fade-in / fade-out 不超过音频一半
  □ video.mp4 分辨率精确 1080 x 1920
  □ 时长 = 音频时长,音画对齐,无黑边
  □ 文件大小符合目标平台上传限制
```

全勾完,才是合格的"封面 + 配音 + 视频"成片。
